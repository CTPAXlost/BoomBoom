# Boom Arena — Android FPS prototype

Самостоятельный мобильный FPS-прототип на **Godot 4.7.1**. Проект не содержит ресурсов, названий, карт или кода Guns of Boom.

## Что реализовано в 0.3

- матч **4 на 4**: игрок + 3 союзных бота против 4 ботов;
- вид от первого лица и простая 3D-арена;
- управление Android: виртуальный стик, свайп для обзора, огонь, перезарядка, нож, 4 слота;
- управление ПК для проверки: `WASD`, мышь, ЛКМ, `R`, `Q`, `1–4`, `F`;
- автострельба по противнику под прицелом, которую можно выключить;
- два основных оружия: автомат **AR-4 Штурм** и дробовик **SG-12 Гром**;
- резервный пистолет при полном окончании патронов основного оружия;
- отдельный удар ножом;
- монеты за убийства и завершение матча;
- покупка дробовика и улучшение урона оружия до 5-го уровня;
- четыре настраиваемых оружейных слота, которые можно оставлять пустыми;
- сохранение монет, покупок, улучшений, снаряжения и автострельбы;
- ограничение 60 FPS;
- GitHub Actions собирает и проверяет подписанный ARM64 APK.

## Сборка через GitHub

1. Создай пустой репозиторий либо замени файлы в существующем `BoomBoom`.
2. Обязательно перенеси скрытую папку `.github`.
3. Для обновления существующего репозитория запусти `ОБНОВИТЬ_GITHUB_0.3.bat`.
4. На GitHub открой **Actions → Build Android APK**.
5. После зелёной сборки скачай artifact `BoomArena-Android-APK`.
6. Внутри будут `BoomArena-debug.apk` и контрольная сумма SHA-256.

## Что исправлено в 0.3

Предыдущий запуск фактически успешно создал и подписал APK, но workflow искал файл не по тому пути. Action поместил APK в `build/Android/BoomArena-debug.apk`, а проверка ожидала `build/BoomArena-debug.apk`.

Теперь workflow автоматически ищет APK внутри `build/`, проверяет его через `apksigner` и `aapt`, создаёт SHA-256 и только после этого загружает artifact.

## Открытие проекта локально

- Установи Godot 4.7.1 Standard.
- Нажми **Import** и выбери `project.godot`.
- Запусти проект кнопкой **F6/F5**.

## Ограничения прототипа

Пока нет сетевого сервера, реальных игроков, качественных моделей и анимаций, звуков, полноценного поиска пути, аккаунтов, античита и серверной экономики. Сетевую часть разумно добавлять после стабилизации локального боя и Android-управления.
